# Internationalization (i18n)

This project uses Python `gettext` files to return API messages in different languages.

## How language is selected

The API checks language in this order:

1. Query parameter: `?lang=fa` or `?lang=en`
2. HTTP header: `Accept-Language: fa`
3. Default language: `en`

Examples:

```text
GET /messages/welcome?lang=fa
GET /messages/welcome
Accept-Language: fa
```

## Translation files

Translation files are stored here:

```text
core/locales/en/LC_MESSAGES/messages.po
core/locales/fa/LC_MESSAGES/messages.po
```

The `.po` files are readable text files. The `.mo` files are compiled files that Python reads at runtime.

## Adding a new language

For example, to add Arabic:

1. Create this folder:

```text
core/locales/ar/LC_MESSAGES/
```

2. Copy `messages.po` from English or Persian into that folder.

3. Translate each `msgstr`.

4. Add the short language code to `SUPPORTED_LANGUAGES` in:

```text
core/core/i18n.py
```

5. Compile the `.po` file into `.mo`.

With gettext installed, the command looks like this:

```text
msgfmt core/locales/ar/LC_MESSAGES/messages.po -o core/locales/ar/LC_MESSAGES/messages.mo
```

## Important notes

Keep message keys simple and consistent. If a text in the code changes, update it in all `.po` files too.

For beginner projects, it is easier to translate response messages first. Translating validation errors from Pydantic is possible, but it needs extra setup and can be added later.
